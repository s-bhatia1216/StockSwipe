# StockSwipe Feature Breakdown

This document describes the features currently implemented in the app. It is intentionally limited to shipped behavior.

## 1. Discover Experience

### 1.1 Swipe Deck and Card Interactions
- Tinder-style stock cards with drag physics.
- Live swipe intent indicators (`BUY` / `SKIP`) while dragging.
- Front/back card flip interaction with smooth 3D transition.
- Floating visual effects during swipes and interactions.

### 1.2 Live Market Data
- Live quote + chart data fetched through backend proxy.
- Time range support: `1D`, `5D`, `3M`, `6M`, `1Y`, `5Y`.
- Correct percentage math by range using Yahoo baseline logic.
- In-memory caching on backend by range-specific TTL.
- Deterministic chart fallback generation when live data is unavailable.

### 1.3 Smart Deck Ordering
- Remaining discover cards are re-ranked using local swipe behavior.
- Learns preference by:
  - Sector affinity (right vs left swipe history).
  - Style affinity (high-vol vs defensive based on beta).
- Ranking is persisted in local storage and evolves continuously.

### 1.4 AI and Context on Card Back
- Static stock summary and bull/bear context.
- "Why this stock?" hook generated by Claude Haiku and cached.
- Ask Claude search bar for freeform stock questions.
- Claude response displayed inline in answer bubble.

### 1.5 News Widgets
- Live company news via Finnhub (through backend).
- 2x2 card grid with headline, source, and relative time.
- Attempts to enrich each story with real `og:image`.
- Graceful fallback if images are missing or news call fails.

### 1.6 Community Layer on Discover
- Bottom-sheet comments for current ticker.
- Likes, replies, verified badges, stance tagging/filtering.
- Floating comment bubbles surfaced in discover view.

## 2. Reels

- Vertical stock reels powered by YouTube Data API.
- Reels are fetched, interleaved across selected stocks, and session-cached.
- Swipe navigation between videos with progress indicator.
- Tapping a stock chip opens the buy modal flow directly.

## 3. Portfolio

### 3.1 Holdings and Valuation
- Holdings persist in local storage.
- Live per-holding pricing and day change updates.
- Invested amount, derived shares, and current value shown per row.

### 3.2 Gesture Actions
- Swipe a holding right to trigger buy-more flow.
- Swipe a holding left to trigger sell flow.
- Animated affordances indicate the pending action.

### 3.3 Trade Modal and Confirmation
- Unified buy/sell modal reused across portfolio and reels.
- Amount entry with share conversion.
- Sell mode supports MAX shortcut.
- Confirmation overlay with action, amount, and share count.

### 3.4 Portfolio Analytics and Gamification
- Diversification donut by sector and allocation percentage.
- Streak tracking with current and longest streak.
- Badge system for specific portfolio events.

## 4. AI Insights Tab

### 4.1 Portfolio Analysis
- Backend call to Claude Sonnet for structured analysis JSON.
- Returned analysis includes:
  - Grade
  - Headline
  - Summary
  - Strengths
  - Risks
  - Recommendations
  - Risk radar note

### 4.2 Risk Radar
- Radar scores for:
  - Concentration
  - Volatility
  - Correlation
- Score generation combines:
  - Sector concentration math
  - Beta-weighted volatility proxy
  - Pairwise correlation estimate from chart-return series
- Visualized as labeled spider chart with axis values.

### 4.3 Flash Card Insight Widgets
- Core insights are rendered as flippable cards.
- Back side explains each widget in plain language:
  - Grade card meaning
  - Sector allocation interpretation
  - Risk radar explanation
  - Strengths/risks interpretation

### 4.4 Recommendation Action Cards
- Recommendation cards can be tapped to flip.
- Swipe right on recommendation opens buy modal for that ticker.
- Swipe left dismisses recommendation from current view.

### 4.5 Resilience
- Supports API base override (`VITE_API_BASE`).
- Fallback request path attempts local backend when proxied path fails.
- Offline mock analysis fallback preserves demo continuity.

## 5. Friends and Social Feed

- Friends feed card stream with recent stock actions.
- Leaderboard showing most-held names among friends.

## 6. Persistence and App State

Persisted user state includes:
- selected sectors
- portfolio holdings
- skipped tickers
- default invest amount
- streak state
- badges
- swipe learning profile

All persisted via local storage and restored on app load.

## 7. Implemented API Endpoints

- `GET /api/stock/:ticker/:range`
- `GET /api/news-insights/:ticker`
- `GET /api/hook/:ticker`
- `POST /api/ask`
- `POST /api/insights`
